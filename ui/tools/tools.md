# Инструменты построения UI (tools)

Вспомогательные модули для декларативного описания, построения и управления деревом виджетов DearPyGui.

## Файлы

### [[tree.py]]

**Основная (актуальная) реализация** — ООП-дерево с узлами `UINode` и парсером `TreeParser`.

- Класс `UINode` — узел дерева, соответствующий одному виджету DPG или контейнеру.
  - Содержит имя (`name`), тип (`TreeTypes`), функции инициализации (`init_func`), построения (`build_func`) и обновления (`update_func`).
  - Хранит дочерние узлы в словаре `childrens`.
  - Метод `init(tag, **kwargs)` вызывает `init_func`, сохраняет полный тег в `self.tag`.
  - Метод `build()` вызывает `build_func` (опционально).
  - Метод `update()` вызывает `update_func` (опционально).
  - Метод `get_update()` возвращает метод `update` для включения в список периодических обновлений.

- Класс `TreeParser` — статические методы для работы с деревом.
  - `tree_init(root, snowball_tag, **extra_params)` — рекурсивно инициализирует все узлы, формируя полные теги и передавая родительский тег через параметр `parent`.
  - `tree_build(root, snowball_tag)` — рекурсивно вызывает `build()` узлов (обычно после инициализации).
  - `get_update_list(root)` — собирает в список все методы `update` узлов (через `get_update()`).
  - `str_format(root)` — возвращает псевдографическое строковое представление дерева.
  - `find(root, find_tag, seniors_tags, last_finded_senior, trace, first, strict)` — гибкий поиск узлов по имени с поддержкой контекстных предков (`seniors_tags`) и режима `strict` (точное следование пути). Возвращает словарь `{полный_путь: узел}`.

- Перечисление `TreeTypes` — типы узлов:
  - `DPG_COMPLEX_COMPONENT` — контейнер (окно, группа, вкладка).
  - `DPG_SIMPLE_COMPONENT` — простой виджет (кнопка, текст).
  - `WIDGET` — кастомный виджет (наследник `BaseWidget`).

**Роль в проекте:** используется в [[../../main.py]] для построения всего интерфейса приложения.
