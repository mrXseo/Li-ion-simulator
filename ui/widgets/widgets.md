# Виджеты пользовательского интерфейса (widgets)

Кастомные компоненты интерфейса, построенные на базе DearPyGui. Каждый виджет инкапсулирует логику создания и обновления группы элементов DPG, а также взаимодействие с объектами симуляции.

## Файлы

### [[base_widget.py]]

**Абстрактный базовый класс `BaseWidget`** — основа для всех виджетов.

- Поля:
  - `tag` — уникальный идентификатор виджета (строка).
  - `simulation_object` — ссылка на объект симуляции (опционально).
  - `parent_tag` — тег родительского контейнера (устанавливается через `set_parent()`).
  - `_built` — флаг, указывающий, был ли уже вызван метод `build()`.
- Методы:
  - `set_parent(parent_tag)` — задаёт родителя.
  - `build()` — создаёт все элементы DPG (должен быть переопределён).
  - `update()` — обновляет содержимое виджета (вызывается периодически). По умолчанию ничего не делает.
- Свойство `is_built` — возвращает `_built`.

**Назначение:** обеспечивает единый интерфейс для всех виджетов, чтобы их можно было встраивать в дерево `UINode` и автоматически обновлять.

### [[history_plot_widget.py]]

**Класс `HistoryPlotWidget`** — линейный график истории значений из `SimulationObject`.

- Принимает:
  - `simulation_object` — объект с методом `get_all_history()`.
  - `y_keys` — список ключей, значения которых отображать (например, `["I_noise"]`).
  - `window_size` — количество отображаемых последних фреймов.
- В методе `build()` создаёт:
  - `dpg.add_plot` с осями.
  - Для каждого `y_key` — `dpg.add_line_series`.
- В `update()`:
  - Берёт историю из `simulation_object.get_all_history()`.
  - Обрезает до `window_size`.
  - Строит массив X = `[-i for i in range(len(history)-1, -1, -1)]` (последняя точка X=0).
  - Обновляет серии через `dpg.set_value()`.

**Примечание:** ось X отображает «кадры назад» (0 — текущий фрейм, -1 — предыдущий и т.д.).

### [[history_histogram_widget.py]]

**Класс `HistoryPlotHistogramWidget`** — гистограмма квантованных значений из истории.

- Зависит от `HistoryPlotWidget` (использует его данные и ось X).
- Параметры:
  - `eps` — шаг квантования (округление до ближайшего кратного `eps`).
  - `eps_range` — диапазон для слайдера `eps`.
- Внутри:
  - Метод `_quantize(value)` — округляет до ближайшего кратного `eps`.
  - Метод `_compute_histogram(data)` — возвращает уровни и частоты.
- Интерфейс:
  - Слайдер для изменения `eps` (вызывает `_update_histogram()`).
  - `dpg.add_plot` с осью X (синхронизируется с осью Y графика истории) и осью Y (частота).
  - `dpg.add_bar_series` для отображения гистограммы.
- В `update()`:
  - Синхронизирует значение слайдера с текущим `eps`.
  - Пересчитывает гистограмму по последним данным.

**Назначение:** позволяет оценить реальное распределение сгенерированного шума.

### [[noise_distribution_widget.py]]

**Класс `NoiseDistributionWidget`** — управление параметрами шума и отображение теоретической плотности нормального распределения.

- Принимает `NoiseGenerator` (объект с полями `sigma` и `bias`).
- Создаёт:
  - Слайдеры для `sigma` и `bias`.
  - График (plot) с линией теоретической PDF.
- Метод `_pdf_gaussian(x, mu, sigma)` — вычисляет значение нормальной плотности.
- Метод `_update_pdf_plot()`:
  - Строит диапазон X от `bias - 3*sigma` до `bias + 3*sigma` (но не уже ±1).
  - Вычисляет PDF и обновляет серию.
- Колбэки слайдеров:
  - `_on_sigma_change` — вызывает `noise_gen.set_parameters(sigma=...)` и обновляет график.
  - `_on_bias_change` — аналогично.
- В `update()` синхронизирует ползунки с текущими значениями из `NoiseGenerator` (на случай изменения из другого места).

**Роль в проекте:** позволяет пользователю в реальном времени менять параметры шума и видеть теоретическое распределение.
