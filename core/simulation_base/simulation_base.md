
# База симуляции (simulation_base)

Модули для управления пошаговым выполнением объектов симуляции, синхронизации данных и хранения истории.

## Файлы

### [[simulation_engine.py]]

**Класс `SimulationEngine`** — центральный движок.

- Регистрирует объекты (`register()`), назначает им уникальные числовые ID.
- Управляет порядком выполнения через `_order` (список ID).
- Выполняет один фрейм: последовательно вызывает `solve_frame()` у всех зарегистрированных объектов (`step_frame()`).
- Предоставляет механизм **скорректированного доступа к истории** через `get_result(target_id, tail_len)`:
  - Учитывает, выполнился ли целевой объект в текущем фрейме или ещё нет.
  - При необходимости автоматически увеличивает `tail_len` на 1, чтобы избежать взятия ещё не готовых данных.
- Поддерживает полный сброс (`reset()`).

**Важные поля:**
- `_objects`, `_order`, `_index_of` — для быстрого доступа к объектам и их позициям.
- `_current_index` — индекс выполняемого в данный момент объекта (‑1, если фрейм не идёт).
- `_frames_count` — общее число завершённых фреймов.

### [[simulation_object.py]]

**Абстрактный класс `SimulationObject`** — основа для всех компонентов симуляции (модели, шумы, фильтры и т.д.).

- При создании автоматически регистрируется в движке (`simulation_engine.register(self)`).
- Хранит ограниченную очередь результатов `_results_history` (размер задаётся в конструкторе).
- Требует переопределения двух методов:
  - `solve_frame()` — выполнить один шаг симуляции, внутри вызвать `_push_result()`.
  - `reset_state()` — сбросить внутреннее состояние (но не историю).
- Предоставляет **синхронизированный доступ к результатам** через `get_frame_result(tail_len)` (использует движок для коррекции времени).
- Прямой доступ к истории (без коррекции) — через `_get_frame_result()` (используется только движком).
- Вспомогательные методы: `clear_history()`, `get_all_history()`, свойство `current_result`.

**Важные замечания:**
- История хранит словари (например, `{"I_noise": 0.123}`).
- Размер истории ограничен, старые фреймы автоматически удаляются.

## Связь с другими модулями

- [[../elements/noise_generator.py]] — конкретная реализация `SimulationObject` (генератор шума).
- Движок используется также для любых будущих объектов (модели батареи, фильтры и т.д.).

## Использование

Обычно создают один экземпляр `SimulationEngine`, затем создают объекты, передавая ему ссылку.  
Пример:

```python
engine = SimulationEngine()
noise = NoiseGenerator(engine, sigma=0.5, bias=0.1)
engine.step_frame()          # выполняет один фрейм для всех объектов
print(noise.current_noise)   # последнее сгенерированное значение
```
